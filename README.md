# ml-churn-data-pipeline

An automated ML pipeline that ingests daily customer churn CSV files, validates and cleans the data, and trains a versioned Random Forest classifier — orchestrated with Prefect.

---

## Overview

The pipeline runs on a schedule (or on-demand) and processes one CSV file per run. Each run produces:

- Pre-clean and post-clean validation reports
- A cleaned, processed CSV (or a rejected CSV if validation fails)
- A versioned trained model artifact (`.pkl`)
- Training metrics as JSON
- A combined validation summary

Day 2 in the demo dataset is intentionally corrupted to demonstrate validation failure and rejection handling.

## Pipeline Flow

```
data/raw/*.csv
     │
     ▼
[Ingest] — skip if already in manifest
     │
     ▼
[Pre-clean validation] — schema, required columns, target column
     │ fail → save to data/rejected/, write report, mark manifest
     ▼
[Clean] — dedup, strip whitespace, parse TotalCharges, add ingestion_date
     │
     ▼
[Post-clean validation] — row count, numeric columns, target integrity
     │ fail → save to data/rejected/, write report, mark manifest
     ▼
[Save processed CSV] → data/processed/
     │
     ▼
[Train Random Forest] → models/<stem>_model.pkl
     │
     ▼
[Save metrics + combined report] → reports/
     │
     ▼
[Mark manifest] — status: success
```

## Quickstart

### 1. Install dependencies

```bash
python -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```

### 2. Add the source dataset

Place `customer_churn_full.csv` in the project root. The file is not committed (see `.gitignore`). The pipeline expects the [Telco Customer Churn dataset](https://www.kaggle.com/datasets/blastchar/telco-customer-churn) schema.

### 3. Prepare demo data

```bash
python -m scripts.prepare_demo_data
```

This splits the source CSV into three daily files in `data/demo_daily_files/` and intentionally corrupts Day 2 to trigger a validation failure during the demo.

### 4. Reset to a clean state (optional)

```bash
bash scripts/reset_demo.sh
```

Clears all runtime artefacts (logs, reports, models, processed CSVs) and re-runs `prepare_demo_data`.

## Running the Pipeline

### Option A — Prefect scheduled flow (recommended for demo)

In one terminal, start the Prefect flow server (polls every 20 seconds):

```bash
python -m orchestration.prefect_flow
```

In a second terminal, publish daily files one by one (also at 20-second intervals):

```bash
python -m scripts.run_demo_publisher
```

The flow will pick up each file automatically and run the full pipeline.

### Option B — Run manually on a specific file

```bash
python -m src.pipeline --input-file customer_churn_2026_05_29.csv
```

### Option C — Run on the next unprocessed file

```bash
python -m src.pipeline
```

## Validation Stages

### Pre-clean (raw schema)

| Check | Description |
|---|---|
| `not_empty` | Dataset has at least one row |
| `required_columns_exist` | All 21 required columns are present |
| `target_column_exists` | `Churn` column is present |
| `target_not_all_missing` | `Churn` is not entirely null |
| `target_values_recognizable` | `Churn` values are `Yes` / `No` only |

Files that fail pre-clean validation are saved to `data/rejected/` and skipped entirely — no cleaning or training occurs.

### Post-clean

| Check | Description |
|---|---|
| `not_empty_after_cleaning` | Dataset is not empty after cleaning |
| `minimum_row_count` | At least 100 rows remain |
| `target_column_exists` | `Churn` is still present |
| `target_has_no_missing_values` | No nulls in `Churn` |
| `target_values_valid` | Only `Yes` / `No` values |
| `target_has_two_classes` | Both classes are represented |
| `tenure_*` | Numeric and non-negative |
| `MonthlyCharges_*` | Numeric and non-negative |
| `TotalCharges_*` | Numeric and non-negative |

## Reports

|---|---|

| `reports/post_clean/` | Post-cleaning validation result per file |

## Model Training

The model is a `RandomForestClassifier` (scikit-learn) with:

- 100 estimators
- `class_weight="balanced"` to handle churn imbalance
- 80/20 stratified train/test split
- One-hot encoding via `pd.get_dummies`

A versioned model is saved per source file:

```
models/customer_churn_2026_05_29_model.pkl
```

Training metrics from the demo runs:

| File | Accuracy | Precision | Recall | F1 |
|---|---|---|---|---|
| `2026_05_29` | 0.8016 | 0.6449 | 0.5308 | 0.5823 |
| `2026_05_31` | 0.7843 | 0.6575 | 0.4324 | 0.5217 |

`2026_05_30` was rejected at pre-clean validation (missing `Churn` column) — no model was trained.

## State Management

`logs/processed_files.json` acts as an idempotency manifest. Each entry records:

```json
{
    "file_name": "customer_churn_2026_05_29.csv",
    "file_path": "/path/to/file",
    "status": "success",
    "training_triggered": true,
    "processed_at_utc": "2026-05-31T08:23:33.689830+00:00"
}
```

Possible statuses: `success`, `failed_pre_clean_validation`, `failed_post_clean_validation`.

The Prefect flow skips any file already present in this manifest, so re-runs are safe.

## Configuration

All paths, column names, and hyperparameters are centralised in `src/config.py`. Key values:

| Constant | Default | Description |
|---|---|---|
| `TARGET_COLUMN` | `Churn` | Name of the label column |
| `MINIMUM_ROW_COUNT` | `100` | Minimum rows after cleaning |
| `TEST_SIZE` | `0.2` | Train/test split ratio |
| `RANDOM_STATE` | `42` | Reproducibility seed |
| `VALID_CHURN_VALUES` | `{"Yes", "No"}` | Accepted target values |
